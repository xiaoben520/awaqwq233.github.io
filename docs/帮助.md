## 一、写博客 & 上传

### 1.1 写一篇新文章

在 `public/blog/posts/` 文件夹下创建 `.html` 文件，例如 `my-post.html`。

**使用下面的模板：**

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>文章标题 · awaqwq233</title>
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&display=swap" rel="stylesheet">
  <style>
    body { font-family: 'Inter', sans-serif; max-width: 720px; margin: 3rem auto; padding: 0 1.5rem; line-height: 1.9; color: #1e293b; }
    h1 { font-size: 2rem; font-weight: 800; margin-bottom: 0.5rem; }
    .meta { color: #94a3b8; font-size: 0.9rem; margin-bottom: 2rem; padding-bottom: 1rem; border-bottom: 1px solid #e2e8f0; }
    p { margin-bottom: 1.25rem; }
    h2 { font-size: 1.4rem; font-weight: 700; margin: 2rem 0 1rem; }
    pre { background: #0f172a; color: #e2e8f0; padding: 1rem 1.25rem; border-radius: 8px; overflow-x: auto; font-size: 0.9rem; }
    code { background: #f1f5f9; padding: 0.15rem 0.4rem; border-radius: 4px; font-size: 0.9em; }
    pre code { background: none; padding: 0; }
    blockquote { border-left: 4px solid #3B82F6; padding: 0.75rem 1.25rem; margin: 1.5rem 0; background: #EFF6FF; border-radius: 0 8px 8px 0; }
    a { color: #3B82F6; text-decoration: none; }
    a:hover { color: #2563EB; }
    .back { margin-top: 3rem; padding-top: 1.5rem; border-top: 1px solid #e2e8f0; }
  </style>
</head>
<body>
  <h1>你的文章标题</h1>
  <div class="meta">
    <span>📅 2026-07-14</span> &nbsp;·&nbsp; <span>💻 技术</span>
  </div>

  <p>这里写你的正文内容...</p>

  <h2>二级标题</h2>
  <p>更多内容...</p>

  <pre><code>代码块在这里</code></pre>

  <blockquote>引用文字</blockquote>

  <div class="back">
    <a href="/#/blog">← 返回博客列表</a>
  </div>
</body>
</html>
```

### 1.2 注册到博客列表

编辑 `public/blog/posts/index.json`，把你的文章信息加进去：

```json
[
  {
    "title": "你的文章标题",
    "date": "2026-07-14",
    "category": "tech",
    "categoryLabel": "💻 技术",
    "tags": ["标签1", "标签2"],
    "excerpt": "会显示在博客列表卡片上的摘要文字",
    "url": "/blog/posts/my-post.html"
  }
]
```

**可选的分类：**

| 分类值 | 显示效果 |
|--------|---------|
| `tech` | 💻 技术（蓝色） |
| `life` | 🌸 生活（粉色） |
| `dev` | ⚙️ 开发（青色） |
| `toy` | 🎮 玩具（黄色） |

> 多篇文章时用逗号分隔，按日期从新到旧排列。

### 1.3 上传部署

```bash
# 添加文件
git add .

# 提交
git commit -m "feat: 添加新博客《文章标题》"

# 推送到 GitHub
git push
```

等 GitHub Actions 跑完（约 1 分钟），网站自动更新 ✅

## 二、修改历程（时间线）

打开 `src/pages/About.jsx`，找到第 23 行左右的 `timeline` 数组：

```jsx
const timeline = [
  { title: '开始搭建个人网站', time: '2025 年 10 月', desc: '使用 GitHub Pages 搭建了个人网站' },
  { title: '武汉大学 · 软件工程', time: '2024 - 至今', desc: '学习软件开发、数据结构与算法等课程' },
  { title: '开始在 GitHub 上活动', time: '2017 年 3 月', desc: '注册 GitHub 账号，开始了开源之旅' },
]
```

**增删改条目：**

- 加新事件 → 在数组里加一行 `{ title, time, desc }`
- 改已有事件 → 直接改对应的值
- 删事件 → 删除对应行（注意逗号）

> 事件会按数组顺序从上到下显示，左侧自动生成蓝色圆点时间线。

## 三、添加小玩具

打开 `src/pages/Toys.jsx`，找到第 17 行的 `toys` 数组：

```jsx
const toys = [
  {
    name: '💓 Heartbeat',
    desc: '一个有趣的心跳互动应用。',
    url: 'https://awaqwq233.com/heartbeat/',
    bg: 'linear-gradient(135deg, #667eea, #764ba2)',
    icon: 'fa-heart',
  },
]
```

**每个玩具的字段说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `name` | 显示名称 | `'💓 Heartbeat'` |
| `desc` | 描述文字 | `'介绍语...'` |
| `url` | 跳转链接 | `'https://xxx.com'` |
| `bg` | 顶部渐变背景色 | `'linear-gradient(135deg, #color1, #color2)'` |
| `icon` | Font Awesome 图标名 | `'fa-heart'` |

**特殊类型（不使用 url 时）：**

- **即将上线**：设置 `url: null` + `soon: true` → 显示灰色 "即将上线" 按钮
- **联系我**：设置 `url: '/#/about'` + `contact: true` → 显示 "联系我" 按钮

## 五、本地预览

在电脑上运行开发服务器，实时查看修改效果：

```bash
# 安装依赖（第一次或新增依赖时运行）
npm install

# 启动开发服务器（会自动打开浏览器）
npm run dev
```

会显示一个本地地址如 `http://localhost:5173`，在浏览器打开即可预览。修改代码后页面会自动刷新。

按 `Ctrl + C` 停止服务器。

## 六、部署上线

### 首次部署

1. 打开 https://github.com/awaqwq23/awaqwq23.github.io/settings/pages
2. **Source** 选择 **GitHub Actions**
3. 回到仓库的 **Actions** 标签页，确认部署工作流成功运行（绿色 ✅）

### 日常更新

修改代码后：

```bash
git add .
git commit -m "描述你改了啥"
git push
```

推送后 GitHub Actions 会自动构建部署，约 1-2 分钟后网站更新。
